# Implementation Plan - Phase 44: Slides Pane Shortcut Keys and UI Navigation Controls

This plan details the addition of shortcut keys, toolbar buttons, bookmark management, and verse shift controls to the slides preview pane.

---

## 1. Simple English Summary of Features (Updated)

### 📖 Bookmarks (Favorites for Slides)
* **What it does**: You can bookmark (tag) any slide in the slides list. Bookmarked slides will show a yellow bookmark icon next to their name.
* **Stable Behavior (No clearing on change)**:
  - Bookmarks are **not** cleared when you change songs or insert new items.
  - To prevent bookmarks from getting lost or pointing to the wrong slide when items are added, removed, or moved, each item in the setlist now gets a stable, unique ID when created.
  - Bookmarks are remembered by matching this stable item ID and the slide's position inside that item. So even if the item shifts in the list, the bookmark moves with it!
* **Buttons**:
  * An **Up Arrow** button to go to the previous bookmarked slide.
  * A **Bookmark** button to toggle the bookmark on or off for the current slide.
  * A **Down Arrow** button to go to the next bookmarked slide.
* **Keyboard Shortcuts**:
  * Press `Ctrl + B` to bookmark/unbookmark a slide.
  * Press `Ctrl + Up Arrow` to jump up to the previous bookmarked slide.
  * Press `Ctrl + Down Arrow` to jump down to the next bookmarked slide.

### 📜 Scripture Search Sync
* **What it does**: If you are displaying a Bible verse, clicking this button will instantly search for that exact verse in the Bible tab and select the correct book, chapter, and verse list options.
* **Focus Behavior**: After executing the search, the text input cursor and focus **will remain in the Bible search textbox**, allowing you to immediately edit or type a new reference.
* **Button**: A **Scripture (Scroll)** icon button in the header.
* **Keyboard Shortcut**: Press `Ctrl + S` while focusing a scripture slide.
* **Behavior**: It automatically switches your view to the Bible tab, types the verse in the search bar, runs the search, highlights the right verse, and keeps focus on the search input box.

### ↔️ Adding Previous/Next Verses & Clarification on Double-Press
* **What it does**: While displaying a Bible verse, you can quickly add the verses before or after it to your presentation setlist.
* **Buttons**:
  * **Double Left Arrow**: Adds the preceding 2 verses (e.g., if you are on John 2:3, it adds John 2:1-2).
  * **Single Left Arrow**: Adds the preceding 1 verse (e.g., if you are on John 2:2, it adds John 2:1).
  * **Single Right Arrow**: Adds the next 1 verse (e.g., if you are on John 2:2, it adds John 2:3).
  * **Double Right Arrow**: Adds the next 2 verses (e.g., if you are on John 2:2, it adds John 2:3-4).
* **Keyboard Shortcuts**:
  * Press `Ctrl + Left Arrow` once to add the previous verse. Press it twice quickly (within half a second) to add the previous two verses instead.
  * Press `Ctrl + Right Arrow` once to add the next verse. Press it twice quickly (within half a second) to add the next two verses instead.
* **Clarification of Double-Press Timing & Logic**:
  * *Step 1*: When you press `Ctrl + Left Arrow` once, the app immediately inserts the single preceding verse (e.g., `John 2:2` from a base of `John 2:3`) and shifts the presentation to it so that the screen updates instantly.
  * *Step 2*: If you press `Ctrl + Left Arrow` again within 500ms, the app treats this as a double-press. However, since the first press already moved the active slide to `John 2:2`, a naive shift would calculate the offset from `John 2:2` (resulting in `John 2:1`).
  * *Solution*: To correctly add the preceding two verses of the *original* slide (`John 2:1-2` from `John 2:3`), the app remembers the original slide reference (`John 2:3`) during the first press. When the second press occurs within 500ms, it uses that saved original reference to calculate `John 2:1-2`, removes the single-verse setlist item created in Step 1, and inserts the double-verse item.

## 2. Technical Details & State Management

1. **State Management**:
   - **Bookmarks**: Stored as a set of unique slide keys `final slideBookmarksProvider = StateProvider<Set<String>>((ref) => {});`.
   - **Unique Slide Keys**: A slide key is computed as `"${item.uniqueId}_$relativeIndex"`, where `relativeIndex` is the slide's offset within the slides generated by its parent setlist item.
   - **Unique Setlist Item IDs**: `SetlistItem` is modified to include a `uniqueId` generated on instantiation. This ensures that bookmarked slides remain correctly mapped to their items regardless of list modifications or swaps.
   - **Search Query Bridge**: A new `final bibleSearchQueryProvider = StateProvider<String?>((ref) => null);` is introduced. When set, the Bible tab listener updates the search controller, executes the search, and resets the provider state to `null` while keeping focus on the search field.
2. **UI layout**:
   - Embedded in the slides pane header:
     - **Bookmark Set** (aligned towards right, but not right-most): `Up Arrow` (`Icons.keyboard_arrow_up_rounded`), `Bookmark` (`Icons.bookmark_border_rounded`/`Icons.bookmark_rounded`), and `Down Arrow` (`Icons.keyboard_arrow_down_rounded`).
     - **Scripture Set** (aligned to right-most): `Double Left Arrow` (`Icons.keyboard_double_arrow_left_rounded`), `Single Left Arrow` (`Icons.keyboard_arrow_left_rounded`), `Scripture` (the scroll icon asset `assets/icons/scroll.png`), `Single Right Arrow` (`Icons.keyboard_arrow_right_rounded`), and `Double Right Arrow` (`Icons.keyboard_double_arrow_right_rounded`).
     - These sets are visually grouped using a dark container matching the theme, with a subtle border and 4px corner radius.
3. **Verse Boundaries**:
   - Shifting verses handles book boundaries using a merged list of books in Genesis-to-Revelation order.
   - Chapter verse counts are queried dynamically from the database to correctly wrap around to the last verse of the previous chapter.

## 4. Verification Plan

### Manual Verification Scenarios
1. **Bookmark Toggling & UI Indication**:
   - Focus the Slides Pane. Navigate to any slide.
   - Click the bookmark icon in the header or press `Ctrl + B`.
   - Verify that the bookmark icon toggles and a yellow bookmark icon is displayed to the left of the slide title.
   - Verify multiple slides can be bookmarked simultaneously.

2. **Bookmark Navigation**:
   - Bookmark slides 2 and 5.
   - Set current slide to 1. Press `Ctrl + Down` or click the down arrow icon. Verify active slide moves to 2.
   - Press `Ctrl + Down` again. Verify active slide moves to 5.
   - Press `Ctrl + Down` again. Verify it stays on 5.
   - Press `Ctrl + Up`. Verify it moves back to 2.
   - Change setlist item to another song or insert items before/after. Verify bookmark indicators remain pointing exactly to the bookmarked slides and navigation still works.

3. **Scripture Sync to Search**:
   - Select a scripture setlist item. Navigate to a slide (e.g. `John 3:16 ESV`).
   - Click the scripture scroll icon or press `Ctrl + S`.
   - Verify the Bible tab opens, the search field gets populated with `John 3:16 ESV`, the correct selections are done, and **focus remains in the Bible search textbox**.

4. **Verse Shifting**:
   - Select a verse slide (e.g. `John 2:2`).
   - Click `Single Left Arrow` or press `Ctrl + Left`. Verify `John 2:1` is added to the setlist and active slide changes to it immediately.
   - Click `Single Right Arrow` or press `Ctrl + Right`. Verify `John 2:2` is added to the setlist and active slide changes to it.

5. **Double Verse Shifting**:
   - From `John 2:3`, click `Double Left Arrow` or press `Ctrl + Left` twice immediately. Verify that `John 2:1-2` is inserted as a single item and the slide updates.
   - Verify the intermediate single-verse setlist item inserted by the first keypress of a double-keypress sequence is cleanly removed, leaving only the 2-verse setlist item.
