Phase 25

Context:
- Extended functionality in slides projection with respect to Monitor settings (max no.of verses, max no.of characters, format)
- This functionality applies to only group of verses slides (eg: John 1:2-5, Gen 3:4-6 etc)
- This functionality does not apply to songs slides and image slides
- This functionality does not apply to single verse slides (eg: Luk 3:5, Rev 5:2 etc)

Interactions:
1) Scenario 1: When Max No. of verses / Slide is '1'. 
- Then the existing functionality remains same, there is no change. you can ignore Max No. of characters / slide and Format.
2) Scenario 2: When Max No. of verses / slide is '2'
- when user selects first verse in the group of verses as of now only that verse is being be displayed. but now the functionality needs to be extended. 
- When user selects first(top) verse in the group of verses
      -- add the immediate next verse and check no of total characters
      -- if the characters count exceeds the no. of max characters settings 
         --- then only the selected verse should be displayed. 
      -- if the characters count does not exceeds the no. of max characters settings 
         --- then display the both verses (slides) at a time in the live screen. 
             ---- if format is 'verse' display next verse in new line
             ---- if format is 'paragraph' display both verses as a single paragraph. 
         --- verse numbers should be displayed always.
         --- when user presses down arrow 
             ---- then move to the next set of verses (in this case 3rd)
             ---- not from first verse to the second verse (because second verse is already combined with first)
         --- same functionality for up arrow as well
         --- when user selects second verse still it should be combined with first verse (the live screen view remains the same).
- so when user selects first or second verse or any verse the combination checking should always flow from top verse and has to determine whether the selected verse should be combined with previous or next or just only that single verse should be displayed. 
3) Scenario 3: When Max No. of verse / slide is '3'
- when user selects first verse in the group of verses as of now only that verse is being be displayed. but now the functionality needs to be extended. 
- When user selects first(top) verse in the group of verses
      -- add the immediate next two verses and check no of total characters
      -- if the characters count exceeds the no. of max characters settings 
         --- then add only the next verse and check no of total characters
         --- if the characters count exceeds the no. of max characters settings 
             ---- then only the selected verse should be displayed. 
         --- if the characters count does not exceeds the no. of max characters settings 
             ---- then display the both verses (slides) at a time in the live screen. 
                  ----- if format is 'verse' display next verse in new line
                  ----- if format is 'paragraph' display both verses as a single paragraph. 
         --- verse numbers should be displayed always.
             ---- when user presses down arrow 
                  ----- then move to the next set of verses (in this case 3rd) 
                  ----- not from first verse to the second verse (because second verse is already combined with first)
             ---- same functionality for up arrow as well
             ---- when user selects second verse still it should be combined with first verse (the live screen view remains the same).
      -- if the characters count does not exceeds the no. of max characters settings 
         --- then display all three verses at same time
	     ----- if format is 'verse' display next verse in new line
             ----- if format is 'paragraph' display both verses as a single paragraph. 
         --- when user presses down arrow
             ---- then move to next set of verses (in this case 4th)
             ---- not to the second or third (because 1st, 2nd and 3rd are already combined and displayed)
         --- same functionality for up arrow as well
- so when user selects first or second or third verse or any verse the combination checking should always flow from top verse and has to determine whether the selected verse should be combined with previous two or previous one or next two or next one or just only that single verse should be displayed. 


Visual Styles:
- When two slides/verses are being displayed at once: then show both the slides as selected
- when three slides/verses are being displayed at once: then show all three slides as selected

Important (note):
- These monitor settings functionality should not affect the preset settings (background, font color, font family, all typography settings). that functionality should remain same. 
- As of now apply the functionality to Monitor 1 only. because if the settings are different for Monitor 1 and Monitor 2 then the same conditions could not be applied. this type of combination we can handle in next phase.
